# Planning การทำงาน: หน้า Check Product (ตรวจสอบการสั่งซื้อสินค้า)

## 4. หน้า Check Product (ตรวจสอบการสั่งซื้อสินค้า)

### 4.1 การแสดงผลหน้าจอ
- แสดง Header ด้านบน ประกอบด้วย:
  - Logo "KIDDYCARE" ด้านซ้าย
  - ไอคอน Search, Shopping Cart (แสดงจำนวนสินค้า), และ User Profile ด้านขวา
- แสดงชื่อหน้า "การสั่งซื้อสินค้า" พร้อมเส้นทาง Breadcrumb: "ร้านค้า / ตรวจสอบการสั่งซื้อสินค้า"

### 4.2 สรุปรายการสินค้าที่สั่งซื้อ
แสดงรายการสินค้าทั้งหมดจากตะกร้า (แบบย่อ) ประกอบด้วย:

#### 4.2.1 การ์ดสินค้าแต่ละรายการ
สำหรับแต่ละสินค้า แสดง:
- รูปภาพสินค้า (ด้านซ้าย ขนาดเล็ก)
- ข้อมูลสินค้า (ด้านขวา):
  - ชื่อหมวดหมู่สินค้า (ตัวเล็ก สีเทา)
  - **ชื่อสินค้า** (ตัวหนา)
  - ตัวเลือกสินค้า (เช่น "ประเภท: สีฟ้าสูตรคลาสสิค")
  - จำนวนสินค้า (เช่น "จำนวน: 1 หน่วย")
- **ราคา** (ด้านขวาสุด ตัวหนา)

**หมายเหตุ**: ในหน้านี้ไม่สามารถแก้ไขจำนวนหรือลบสินค้าได้ เป็นเพียงการแสดงผลเพื่อยืนยัน

### 4.3 สรุปยอดเงิน
แสดงในกรอบสรุป:
- **ยอดรวม**: ราคารวมทุกสินค้า (฿ 1493.00)
- **ค่าจัดส่ง (ปั้นดิน)**: ค่าจัดส่ง (฿ 50.00)
- **รวมทั้งหมด**: ยอดรวมสุทธิที่ต้องชำระ (฿ 1543.00) - **ตัวหนา ขนาดใหญ่**

### 4.4 ฟอร์มตรวจสอบการสั่งซื้อสินค้า

#### 4.4.1 คำอธิบาย
แสดงข้อความอธิบาย:
```
"เมื่อตรวจสอบรายการสินค้าเรียบร้อยแล้ว กรุณายืนยันอีเมลล์
เราจะส่งการยืนเตสถานะในสิ่งซื้อให้ทางที่อยู่เมลล์หลักสะดุโว้
ในบัญชีของคุณ เปลี่ยนอีเมลล์"
```
- มีลิงก์ **"เปลี่ยนอีเมลล์"** (สีชมพู underline) สำหรับเปลี่ยนอีเมลล์รับการแจ้งเตือน

#### 4.4.2 ช่องกรอกอีเมลล์
- Label: **"อีเมลล์*"** (มีเครื่องหมาย * บ่งบอกว่าเป็น required field)
- Input field: กรอกอีเมลล์ผู้ใช้
  - Placeholder: "กรุณาใส่อีเมลล์ของคุณ"
  - ดึงอีเมลล์จากข้อมูลผู้ใช้ที่ login มาแสดงเป็นค่าเริ่มต้น (pre-filled)
  - สามารถแก้ไขได้

#### 4.4.3 Checkbox ยืนยันเงื่อนไข
- Checkbox: "ฉันได้ทำการตรวจสอบรายการสินค้าเรียบร้อยแล้ว"
- **ต้องติ๊กถูกก่อนจึงจะสามารถกดปุ่ม "ดำเนินการต่อ" ได้**

### 4.5 Validation และ Error Handling

#### 4.5.1 การตรวจสอบอีเมลล์
- ถ้าช่องอีเมลล์ว่าง:
  - แสดงข้อความเตือน "กรุณากรอกอีเมลล์" เป็น text สีแดงใต้ช่อง input
  
- ถ้าอีเมลล์ไม่ถูกรูปแบบ (ไม่มี @ หรือรูปแบบไม่ถูกต้อง):
  - แสดงข้อความเตือน "รูปแบบอีเมลล์ไม่ถูกต้อง กรุณากรอกใหม่" เป็น text สีแดงใต้ช่อง input

#### 4.5.2 การตรวจสอบ Checkbox
- ถ้ายังไม่ติ๊ก Checkbox:
  - ปุ่ม "ดำเนินการต่อ" จะเป็น disabled (สีเทา ไม่สามารถกดได้)
  - เมื่อพยายามกด จะแสดงข้อความเตือน "กรุณายืนยันการตรวจสอบรายการสินค้า" เป็น text สีแดง

- ถ้าติ๊ก Checkbox แล้ว:
  - ปุ่ม "ดำเนินการต่อ" จะเป็น active (สีชมพูเข้ม กดได้)

### 4.6 ปุ่มดำเนินการ

#### 4.6.1 ปุ่ม "แก้ไขรายการสินค้า"
- ขอบสีชมพู พื้นหลังสีขาว
- เมื่อกด:
  - นำกลับไปยังหน้า Bucket List Summation (ตะกร้าสินค้า)
  - เพื่อแก้ไขจำนวนสินค้า เปลี่ยนตัวเลือก หรือลบสินค้า

#### 4.6.2 ปุ่ม "ดำเนินการต่อ"
- พื้นหลังสีชมพูเข้ม ตัวอักษรสีขาว
- เมื่อกด:
  1. **ตรวจสอบ Validation ทั้งหมด**:
     - ช่องอีเมลล์ต้องไม่ว่าง
     - อีเมลล์ต้องมีรูปแบบถูกต้อง
     - Checkbox ต้องถูกติ๊ก
  
  2. **ถ้า Validation ผ่าน**:
     - บันทึกอีเมลล์ที่กรอกลงฐานข้อมูล (ถ้ามีการเปลี่ยนแปลง)
     - บันทึกสถานะว่าผู้ใช้ได้ยืนยันการตรวจสอบแล้ว
     - นำไปยังหน้า Payment (หน้าชำระเงิน) หรือหน้า Shipping Information (กรอกที่อยู่จัดส่ง) ขึ้นอยู่กับ flow ของระบบ
  
  3. **ถ้า Validation ไม่ผ่าน**:
     - แสดงข้อความเตือนที่เกี่ยวข้องเป็นสีแดง
     - ไม่ให้ดำเนินการต่อ

### 4.7 ฟีเจอร์ "เปลี่ยนอีเมลล์"
- เมื่อกดลิงก์ "เปลี่ยนอีเมลล์":
  - เปิด Modal หรือ expand form เพื่อให้แก้ไขอีเมลล์ได้
  - สามารถกรอกอีเมลล์ใหม่และยืนยัน
  - บันทึกอีเมลล์ใหม่ลงฐานข้อมูลผู้ใช้
  - แสดงข้อความยืนยัน "เปลี่ยนอีเมลล์สำเร็จ" เป็นสีเขียว

### 4.8 ขั้นตอนต่อไป (Next Steps)
หลังจากกดปุ่ม "ดำเนินการต่อ" แล้ว ระบบจะนำไปยัง:
1. **หน้ากรอกที่อยู่จัดส่ง** (Shipping Information Page) หรือ
2. **หน้าเลือกวิธีชำระเงิน** (Payment Method Page)

ขึ้นอยู่กับว่าระบบมีข้อมูลที่อยู่จัดส่งของผู้ใช้อยู่แล้วหรือไม่

### 4.9 ส่วนขั้นตอนถัดไป (ด้านล่างสุด)

#### 4.9.1 แสดงไอคอน 3 ขั้นตอน
พร้อมคำอธิบาย:

1. **วิธีการจัดส่ง** (ไอคอนรถส่งของสีเขียว)
   - "เลือกวิธีที่คุณต้องการรับสินค้า"

2. **ข้อมูลการชำระเงิน** (ไอคอนเงินสีน้ำเงิน)
   - "เลือกวิธีการชำระเงินและชำระเงินดำเนินการชำระเงิน"

3. **ยืนยันการสั่งซื้อ** (ไอคอนเอกสารสีชมพู)
   - "รับรายการสั่งซื้อและรับรองยืนยันสั่น"

**หมายเหตุ**: ส่วนนี้แสดงให้ผู้ใช้เห็นภาพรวมของขั้นตอนการสั่งซื้อทั้งหมด

### 4.10 การเชื่อมต่อฐานข้อมูล Sevalla
- ดึงข้อมูลสินค้าทั้งหมดจากตะกร้าของผู้ใช้
- ดึงข้อมูลอีเมลล์ผู้ใช้จาก User Profile
- บันทึกอีเมลล์ที่กรอก (ถ้ามีการเปลี่ยนแปลง)
- บันทึกสถานะว่าผ่านหน้าตรวจสอบแล้ว (Checkpoint)
- เตรียมข้อมูลสำหรับหน้าถัดไป (Payment/Shipping)

### 4.11 UX/UI Notes
- ใช้สีชมพู (#FF1493) เป็นสีหลัก
- ปุ่ม "ดำเนินการต่อ" จะเป็น disabled จนกว่าจะติ๊ก Checkbox
- แสดง Loading indicator เมื่อกำลังบันทึกข้อมูล
- ใช้ไอคอนที่สื่อความหมายชัดเจนสำหรับขั้นตอนต่างๆ
- รองรับ Responsive Design สำหรับ Mobile และ Desktop
- แสดง Error message ใกล้กับช่อง input ที่เกิด error
- ใช้สี transition/animation เมื่อเปลี่ยนสถานะปุ่ม disabled → active

### 4.12 Security & Privacy
- ไม่แสดงอีเมลล์เต็มรูปแบบ สามารถแสดงแบบ m***@example.com เพื่อความปลอดภัย (optional)
- ตรวจสอบ Session ว่าผู้ใช้ยัง login อยู่หรือไม่
- ป้องกัน Cross-site Scripting (XSS) ในช่อง input
